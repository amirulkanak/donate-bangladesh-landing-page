Important FAQ

What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of a webpage as a tree of objects, allowing scripts (like JavaScript) to dynamically access and manipulate the content, structure, and styling of the document. Each element, attribute, and piece of text in the HTML is treated as a node in this tree, enabling real-time updates to the page without needing to reload it.

How do you select an element from the DOM?

We can select an element from the DOM using JavaScript methods like
getElementById() (e.g., document.getElementById('elementID');),
getElementsByClassName() (e.g., document.getElementsByClassName('className');),
querySelector() (e.g., document.querySelector('.className');) or
querySelectorAll() (e.g., document.querySelectorAll('.className');), depending on whether we are targeting an ID, class, tag, or CSS selector.

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique where a single event listener is added to a parent element to manage events for its child elements. It's useful because it improves performance, reduces the number of event listeners, and handles dynamically added elements efficiently.

How do you manipulate an element's attributes and styles using the DOM?

We can manipulate an element's attributes using setAttribute() (e.g., element.setAttribute('src', 'image.jpg');), getAttribute() (e.g., element.getAttribute('href');), or directly by accessing the attribute (e.g., element.id= 'new-id';). To change styles, we can use the style property (e.g., element.style.color = "blue";).
